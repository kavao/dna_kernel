## 目的

「資料が揃っているつもりで書き始めたら途中で詰まった」を防ぐ。
作業着手の**前に**、必須の成果物・ディレクトリが存在することを機械的に確認する。

## 基本パターン

作業対象フォルダに対してチェックスクリプトを実行し、終了コード **0** を確認してから本作業に入る。
**1** なら欠けているものを作ってから再チェックする。

```bash
python tools/<プロジェクト固有のチェックスクリプト> <作業対象フォルダ>
```

チェックスクリプトの作り方は「プロジェクトへの適用」を参照。

### 小説プロジェクトでの実装例

`tools/kernel/novel_project_check.py` は、このパターンを小説プロジェクト向けに実装したものです。
作品フォルダに必要なファイル・ディレクトリが揃っているかを確認します。

```bash
python tools/novel_project_check.py <作品フォルダ>
```

## プロジェクトへの適用

このスキルのパターンを別プロジェクトへ持ち込む場合、次の3点を定義すれば動く。

### 1. 必須ファイルの一覧

作業開始の前提として存在すべきファイルを列挙する。

```python
REQUIRED_FILES = (
    "design.md",     # 設計書
    "config.md",     # 基本設定
    "README.md",     # 概要
)
```

### 2. 必須ディレクトリの一覧

空でよいが存在すべきディレクトリを列挙する。

```python
REQUIRED_DIRS = (
    "output/",   # 成果物置き場
    "review/",   # レビュー結果置き場
)
```

### 3. チェックスクリプト or チェックリスト

- **スクリプトで自動化**（推奨）: `tools/kernel/novel_project_check.py` を参考に `REQUIRED_FILES` と `REQUIRED_DIRS` を書き換えて使う。終了コードを LLM に確認させる。
- **チェックリストで手動**（最小構成）: スキルの `必須（作業前）` 節に箇条書きで書き、LLM に読ませる。

## 必須（作業前）

作業着手の直前に確認する。

- 必須ファイルが全て存在し、空でないこと
- 必須ディレクトリが存在すること（空でよい）
- 設定ファイルに ID やコードがある場合、フォルダ名との整合が取れていること

## 不足時の典型対処

| 不足の種類 | 対処 |
|-----------|------|
| 必須ファイルが無い | 作業前に作成する（空ファイルは不可。最低限の内容を入れる） |
| 必須ディレクトリが無い | `mkdir` または `.gitkeep` 付きで作成する |
| ID / コードの不整合 | 設定ファイルとフォルダ名を揃えてから再チェック |
