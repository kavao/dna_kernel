## 目的

既存ファイルを直接上書きすると**元の内容に戻せなくなる**。
このスキルは、上書きの前に旧版を必ず退避し、「やり直せる状態」を保つ手順を固定する。

## 手順

1. **対象ファイルを特定する**  
   上書き対象のファイルパスを確認する。

2. **バックアップ先を決める**  
   プロジェクトで `_backup/`、`_old/`、`_archive/` など退避フォルダを統一する。  
   フォルダが無ければ作成する。

3. **バックアップファイル名を採番する**  
   `<元ファイル名>_v001.<拡張子>` → `_v002.<拡張子>` のように3桁ゼロ埋めの連番を使う。  
   同一ファイルの既存バックアップ最大番号 + 1 を次版とする。

   ```
   例:
     report.md → _backup/report_v001.md（1回目）
     report.md → _backup/report_v002.md（2回目）
   ```

4. **旧版を退避する**（上書きの前に必ず実施）  
   バックアップパスへコピーまたは移動する。**Read で退避先を確認してから**次へ進む。

5. **正本ファイルを更新する**  
   元と同一のファイル名で上書きする（`output-discipline` に従い、更新後に確認する）。

6. **確認する**  
   - 退避先にバックアップが存在することを確認する。
   - 正本ファイルが期待どおり更新されていることを確認する。

## 宣言のみで終えない

「退避してから更新します」と述べたら、**その応答の中**で退避まで実行する。
宣言だけを出して次の応答に持ち越さない。

## プロジェクトへの適用

2点だけプロジェクト側で決める。

| 設定項目 | 例 |
|----------|-----|
| バックアップフォルダの名前 | `_backup/`、`_old/`、`versions/` |
| 版番号の桁数・形式 | `v001`（3桁）、`20260506`（日付）など |

それ以外の手順（採番 → 退避 → 更新 → 確認）はこのスキルで固定する。

## 禁止

- 旧版退避を省略したまま上書きしない。
- 退避を「確認していないが実施した」と扱わない（Read で存在確認するまで完了ではない）。
